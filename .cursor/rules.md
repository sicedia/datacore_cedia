# datacore_cedia project rules

## Focus

- This project is for **modeling and documentation**, not execution.
- Prioritize **conceptual clarity** over perfect SQL.

## Names and relationships

- Do not assume definitive names: everything is **iterative**.
- Ask before inventing relationships that are not explicit.

## Diagrams and explanations

- Prefer **Mermaid** for diagrams.
- Explain modeling decisions in **business language**.

## Response style

- Keep responses **concise** and **structured**.

## Tools

- Do not introduce tools (dbt, Spark, Airflow) unless explicitly requested.
